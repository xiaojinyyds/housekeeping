## 📋 API接口测试

### 1. 发送验证码
```http
POST http://localhost:8000/api/v1/auth/send-code
Content-Type: application/json

{
  "email": "test@example.com"
}
```

## 🔧 常见问题

### 1. 数据库连接失败
- 检查数据库服务器是否可访问
- 确认用户名密码是否正确
- 检查防火墙设置

### 2. Redis连接失败
- 检查Redis服务是否启动
- 确认密码是否正确
- 检查端口是否开放

### 3. 邮件发送失败
- 检查SMTP配置是否正确
- 确认授权码是否有效
- 查看控制台错误日志

### 4. 端口被占用
如果8000端口被占用，可以修改 `run.py` 中的端口号：
```python
uvicorn.run(
    "app.main:app",
    host="0.0.0.0",
    port=8001,  # 改成其他端口
    reload=True
)
```

## 📊 数据库表结构

系统包含以下核心表：
1. **users** - 用户表（支持三种角色）
2. **worker_applications** - 家政阿姨申请表
3. **worker_profiles** - 家政阿姨详细信息表
4. **services** - 服务项目表
5. **appointments** - 预约订单表
6. **announcements** - 公告表
7. **time_slots** - 时间段配置表
8. **favorites** - 收藏表
9. **browse_history** - 浏览历史表

详细表结构见 `database_design.sql`
