## 一、服务器环境准备

### 1.1 宝塔面板安装软件

在宝塔面板「软件商店」安装：
- Nginx（推荐 1.24+）
- Python项目管理器 3.0
- PM2管理器（可选，用于进程管理）

### 1.2 安装 Python 3.10+

```bash
# 在宝塔终端执行
cd /www/server
wget https://www.python.org/ftp/python/3.10.13/Python-3.10.13.tgz
tar -xvf Python-3.10.13.tgz
cd Python-3.10.13
./configure --prefix=/www/server/python310
make && make install

# 创建软链接
ln -sf /www/server/python310/bin/python3.10 /usr/bin/python3.10
ln -sf /www/server/python310/bin/pip3.10 /usr/bin/pip3.10
```

或者直接在宝塔「Python项目管理器」中安装 Python 3.10 版本。

### 1.3 安装 Node.js 18+

在宝塔「软件商店」→「运行环境」安装 Node.js 18.x 或 20.x

### 2.4 创建启动脚本

创建 `/www/wwwroot/housekeeping/backend/start.sh`：

```bash
#!/bin/bash
cd /www/wwwroot/housekeeping/backend
source venv/bin/activate
exec uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
```

赋予执行权限：
```bash
chmod +x start.sh
```

### 2.5 使用 Supervisor 管理进程

在宝塔「软件商店」安装「Supervisor管理器」，然后添加守护进程：

- **名称**: housekeeping-api
- **启动命令**: `/www/wwwroot/housekeeping/backend/start.sh`
- **运行目录**: `/www/wwwroot/housekeeping/backend`
- **启动用户**: root

或者使用命令行配置：

```bash
# 创建 supervisor 配置
cat > /etc/supervisor/conf.d/housekeeping.conf << EOF
[program:housekeeping-api]
command=/www/wwwroot/housekeeping/backend/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
directory=/www/wwwroot/housekeeping/backend
user=root
autostart=true
autorestart=true
stderr_logfile=/www/wwwroot/housekeeping/backend/logs/error.log
stdout_logfile=/www/wwwroot/housekeeping/backend/logs/access.log
EOF

# 重载配置
supervisorctl reread
supervisorctl update
supervisorctl start housekeeping-api
```

## 三、前端部署

### 3.1 本地构建

在本地项目目录执行：

```bash
cd H5-project/front

# 修改生产环境配置
# 编辑 .env.production
```

### 3.2 修改前端配置

编辑 `H5-project/front/.env.production`：

```env
# 页面标题
VITE_APP_TITLE = 家政管理系统
# 接口请求地址
VITE_APP_API_BASEURL = http://101.43.51.104:8000
# 调试工具（生产环境留空）
VITE_APP_DEBUG_TOOL =
```

### 3.3 构建项目

```bash
# 安装依赖
pnpm install

# 构建
pnpm build
```

构建完成后，`dist` 目录就是要部署的静态文件。

### 3.4 上传到服务器

将 `dist` 目录内容上传到：
```
/www/wwwroot/housekeeping/front
```

### 3.5 配置 Nginx

在宝塔面板「网站」→「添加站点」：
- **域名**: 101.43.51.104（或你的域名）
- **根目录**: /www/wwwroot/housekeeping/front
- **PHP版本**: 纯静态

然后修改 Nginx 配置：

```nginx
server {
    listen 80;
    server_name 101.43.51.104;  # 或你的域名
    
    root /www/wwwroot/housekeeping/front;
    index index.html;
    
    # 前端路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket 支持（如果需要）
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
    
    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
    
    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
    gzip_min_length 1000;
}
```

如果使用反向代理，前端配置可以改为：

```env
# .env.production
VITE_APP_API_BASEURL = /api
```

## 六、常见问题

### Q1: 后端启动失败
```bash
# 查看日志
tail -f /www/wwwroot/housekeeping/backend/logs/error.log

# 手动测试启动
cd /www/wwwroot/housekeeping/backend
source venv/bin/activate
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### Q2: 数据库连接失败
- 检查数据库服务器防火墙是否放行 3306 端口
- 检查数据库用户是否允许远程连接
- 检查 .env 中的数据库配置

### Q3: Redis 连接失败
- 检查 Redis 服务器防火墙是否放行 6379 端口
- 检查 Redis 是否配置了密码认证
- 检查 .env 中的 Redis 配置

### Q4: 前端页面空白
- 检查 Nginx 配置中的 `try_files`
- 检查浏览器控制台是否有 JS 错误
- 检查 API 地址是否正确

### Q5: CORS 跨域错误
- 检查后端 .env 中的 `ALLOWED_ORIGINS` 是否包含前端地址
- 如果使用 Nginx 反向代理，前端应该使用 `/api` 而不是完整地址
